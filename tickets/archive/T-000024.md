## Goal
Standardize on uv for Python dependency management and execution across muonevals, muontickets, and muonledger to avoid Xcode/Homebrew Python conflicts

## Acceptance Criteria
- [ ] `pyproject.toml` for muonevals with dependencies (pytest, muonledger, muontickets)
- [ ] `uv sync` creates a working venv with correct Python version
- [ ] `uv run pytest` passes all tests
- [ ] `uv run python run_autoresearch.py` works
- [ ] Makefile updated to use `uv run` instead of bare `python3`
- [ ] muonledger and muontickets importable as path dependencies via pyproject.toml

## Notes

### Problem
`python3` resolves to different interpreters depending on shell context:
- Terminal: Xcode Python 3.9 (`/Applications/Xcode.app/.../python3`)
- Claude Code: Homebrew Python 3.14 (`/opt/homebrew/opt/python@3.14/bin/python3`)

This causes confusion (xctest process names, missing packages, version mismatches).

### Solution
Use `uv` to pin Python version, manage dependencies, and run all commands:
- `uv run pytest` instead of `python3 -m pytest`
- `uv run python run_autoresearch.py` instead of `python3 run_autoresearch.py`
- Consistent Python version across all environments

### Scope
- **muonevals**: pyproject.toml + uv.lock, Makefile → `uv run`
- **muonledger**: ensure importable as a path dependency
- **muontickets**: ensure `mt.py` works via `uv run`
